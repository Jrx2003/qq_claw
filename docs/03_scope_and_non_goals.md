# 03 范围与非目标

## P0 必做范围

### 核心场景
宿舍群 / 熟人小群的“周五烤肉局”

### 核心页面 / 状态
1. 群聊启动前的自然聊天
2. 虾局长成局卡
3. 虾局长投票进展卡
4. 虾局长确认成局卡
5. 虾局长回忆卡

### 核心能力
- 群内意图被触发
- 虾局长代替用户发起收口
- 时间 / 地点投票
- 未回复成员提示
- 成局确认
- 局后回忆沉淀

### 必须完成的交互
- 评委可以点击一个“触发虾局长”的建议项
- 评委可以点击投票 / 参加 / 提醒 / 查看详情等按钮
- 系统可以自动插入 NPC 消息
- 卡片状态可以更新
- 支持一次完整流程跑通

## P0.5 推荐完成
- 自动播放
- 场景切换器
- 顶部“演示进度条”
- 隐藏调试面板
- 动画与微交互

## P1 扩展
### 扩展 1：匿名倡议
适合表达：
- 想吃饭但不想出头
- 想提建议但怕尴尬
- 想吐槽但不想被点名

### 扩展 2：冲突中继
适合展示：
- 群里争执升级时，虾局长以中性口吻转述诉求
- 回顾历史聊天
- 给出“冷静选项”和“温和表达建议”

### 扩展 3：腾讯游戏局模板
适合展示：
- 王者荣耀开黑局
- 游戏结束后的高光回忆卡

## 非目标
### 不做真实接入
P0 不以真实 QQ API 接入为主目标。

### 不做开放式自由聊天
P0 不让评委无限输入自由文本来探索。

### 不做完整后台
只做极轻量调试控制，不做复杂运营后台。

### 不做真正多用户同步
可以通过本地状态 / 脚本模拟多用户效果。

### 不做真实推荐系统
成员偏好可以使用预设脚本 / JSON 假数据。

## 砍需求优先级
当时间不够时，按以下顺序砍：
1. 先砍匿名倡议
2. 再砍冲突中继
3. 再砍游戏局扩展
4. 再砍调试后台
5. 最后才砍主路径动画

绝对不能砍的：
- 成局卡
- 投票卡
- 确认卡
- 回忆卡
- 主闭环稳定性
