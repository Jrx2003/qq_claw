# 当前仓库与最终目标的差距说明

这份说明是给 Codex 看的。

## 当前已有基础
- Next.js + TypeScript + Tailwind 风格项目底盘
- 基础群聊原型
- 主线 fixtures
- demo page 与 engine 雏形

## 当前主要差距
1. 当前文档与目标不一致：旧文档仍把匿名倡议、冲突桥梁、游戏局写成后续扩展，而不是正式功能。
2. 当前引擎还偏 step/append 模型，不足以支撑录屏导演节奏与 live LLM 插入。
3. 当前 autoplay 还不是“录屏可用”的导演系统。
4. 当前 mock 比重过高，缺少真实 LLM + snapshot 机制。
5. 当前仓库还没有 submission 目录与比赛材料产出体系。
6. 当前部署、环境变量、studio 权限控制都还不成体系。

## 新标准
从现在起，所有开发以“完整参赛项目”为准，不以“最小功能先跑”为准。
