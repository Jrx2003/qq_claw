# 13 验收清单

## A. 产品理解
- [ ] 看完首页，能在 10 秒内理解“虾局长是干什么的”
- [ ] 看完主流程，能在 1 分钟内理解其核心价值
- [ ] 主路径明确是“收口—投票—成局—回忆”

## B. 演示完整性
- [ ] 可以从第一幕完整走到回忆卡
- [ ] 不需要自由输入也能完成演示
- [ ] NPC 会自动发言
- [ ] 卡片状态会跟随剧情变化
- [ ] 支持 Replay
- [ ] 支持 Auto Play

## C. 视觉完成度
- [ ] 整体像 QQ 群聊而不是网页后台
- [ ] 卡片层次清楚
- [ ] 按钮像“可点的产品控件”
- [ ] 中文文案自然
- [ ] 虾局长有独特识别度

## D. AI 原生性感知
- [ ] 用户能感知虾局长在“替人承担组织成本”
- [ ] 用户能感知它不是普通投票器
- [ ] 用户能感知回忆卡是差异化价值
- [ ] 匿名倡议 / 冲突中继 至少在文档或扩展场景里被体现

## E. 技术实现
- [ ] 场景脚本从独立 fixture 读取
- [ ] 业务逻辑没有写死在 UI 里
- [ ] 代码可以被继续扩展
- [ ] 具备未来接真实 QQ / LLM 的接口抽象
- [ ] 没有为了真实接入而让 P0 变脆弱

## F. 比赛表达
- [ ] 产品故事线清晰
- [ ] 可以直接拿去录 3 分钟视频
- [ ] 可以配合说明文档讲评分点
- [ ] Demo 中能体现用户洞察、创新性、AI 原生性和完整性

## G. 录屏适配
- [ ] 主流程总时长控制在 120~180 秒
- [ ] 每一幕都可以停住给口播讲解
- [ ] 有一键自动播放模式
- [ ] 有至少 4 个漂亮的关键画面
