# 14 最终目标总说明（Master Spec）

## 1. 项目北极星

虾局长不是一个“会发几张卡片的投票插件”，而是 **QQ 养虾官方 Agent 的产品化原型**：
它要承担原来由人类群主、活跃成员、组织者承担的那段“最碎、最累、最容易背锅”的社交推进成本。

这意味着虾局长必须同时具备四种产品气质：

1. **官方性**：它像 QQ 里的官方 Agent，不是第三方工具。
2. **社交性**：它围绕群关系和互动推进，而不是孤立效率工具。
3. **AI 原生性**：把 AI 拿掉，核心能力会明显坍塌。
4. **可演示性**：评委在短时间内能看到清楚、完整、可信的产品故事。

## 2. 评委要看到的不是“功能点列表”，而是完整产品论证

所以最终项目必须同时满足：

### 赛道适配
- 明确是题目 3：QQ 养虾 / 社交新玩法
- 功能与 QQ 群聊氛围强绑定
- 不是通用聊天机器人套壳

### 完整性
- 每个核心能力都有最少一个可跑通、可讲解的完整闭环
- 主线必须可以从头走到尾
- 支线必须可以从入口走到结果，而不是只有 teaser

### 创新性
- 真正的创新点不是“AI 帮我总结一下”
- 而是：虾局长承担组织责任、匿名代发、冲突桥接、活动记忆

### 用户洞察
- 不是“年轻人需要陪伴”这种泛洞察
- 而是：
  - 不想当组织者
  - 不想暴露自己发起
  - 不知道怎么把“我都可以”变成决策
  - 吵架时群管理的手段太粗
  - 群活动结束后没有沉淀

### AI 原生性
必须在演示中清楚证明：
- 如果没有 AI，组局建议、匿名改写、冲突桥梁、回忆卡都会退化为低价值模板

## 3. 最终产品结构

## 3.1 核心能力模块

### A. 智能收口组局
适用于：
- 吃饭
- 开黑
- 出门
- 社团活动
- 游戏局

核心动作：
- 识别聊天中的隐性组局意图
- 生成成局建议
- 推动时间/地点投票
- 识别未回复成员
- 自动给出确认成局卡
- 自动生成回忆卡

### B. 匿名倡议
适用于：
- 想提议但不想暴露自己
- 想发起活动但不想背组织责任
- 想表达群内想法但怕尴尬

核心动作：
- 用户把意图委托给虾局长
- 虾局长用官方口吻在群里代发
- 群内看到的是“虾局长倡议”
- 发起人身份对群成员隐藏

### C. 冲突桥梁
适用于：
- 争执升级
- 阴阳怪气
- 反复拉扯
- 群主传统禁言手段过于粗暴

核心动作：
- 回顾最近冲突上下文
- 提炼双方诉求
- 温和中继对方真实想表达的内容
- 提供几种降温选项：
  - 冷静 10 分钟
  - 继续私聊
  - 让虾局长代为转述
  - 回到当前需要决策的问题

### D. 活动与游戏回忆
适用于：
- 宿舍聚餐
- 王者荣耀五排
- 洛克王国组队活动
- 观影 / 出游 / 社团活动

核心动作：
- 自动生成回忆卡
- 提炼高光 / 关键词 / MVP / 表情包级文案
- 提供“下次再约”的快捷入口
- 复用上次偏好

## 3.2 体验壳子

### Judge Mode
目标：
- 10 秒看懂
- 1 分钟感知价值
- 3 分钟看完闭环

要求：
- UI 极简
- 引导强
- 不让评委迷路
- 能轻松切到扩展能力

### Recording Mode
目标：
- 能稳定录官方提交视频
- 自动跑，但在讲解点停下来

要求：
- 一键自动演示
- 卡点暂停
- 文案与节奏可控
- 隐藏 dev UI

### Studio Mode
目标：
- 开发、联调、现场兜底

要求：
- scene switch
- mock/live/snapshot 切换
- replay
- state inspect
- save live output

## 4. 所有功能都要有“导演版本”和“真实版本”

这是本项目最重要的设计原则之一。

### 导演版本
给评委看的稳定体验：
- 强引导
- 节奏可控
- 可以录屏
- 可以确保讲到重点

### 真实版本
给产品可信度加分：
- 接真实 LLM
- 允许有限输入
- 有更接近真实系统的体验

这两个版本不能互相替代，而是要共存。
正确做法是：
- 主路径用导演版本保证稳
- 局部能力用真实版本体现 AI 不是写死的

## 5. 最终成功标准

项目最终必须达到下面这组标准：

### 对评委
- 能快速理解问题
- 能清楚看到 AI 在核心环节起作用
- 能体验到一个像产品而不是像 PPT 的原型

### 对演示
- 能一键跑通完整主线
- 能稳定录出 3 分钟视频
- 支线可在 30~60 秒内切出来单独展示

### 对技术
- 真实 LLM 可用
- 失败有 fallback
- 对外可访问
- 代码结构能继续扩展

### 对比赛材料
- Demo URL 可交
- PDF 素材能从仓库直接整理
- 录屏脚本和截图清单已内置

## 6. 最重要的边界

不要把项目做成：
- 一个自由聊天机器人
- 一个只有吃饭局的单点工具
- 一个只能 mock 的样片
- 一个功能很多但评委不会走的复杂产品

要把它做成：
**一个功能完整、导演有序、AI 可证、可以上线访问、能支撑比赛评审的产品原型。**
