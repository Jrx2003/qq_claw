# 15 功能矩阵与触发规则

## 1. 先明确：触发不是单一规则，而是一套策略系统

你之前提到的关键问题非常对：
- 什么情况下虾局长自动提示？
- 什么情况下必须 @？
- 什么情况下可以直接触发，无需用户操作？
- 某个功能是谁触发的，要不要显示？

这部分不能继续含糊，必须做成 **Trigger Policy**。

## 2. 触发策略总框架

项目中必须存在一套管理员可切换的策略预设：

### Preset A：保守模式
- 只有用户主动点 chip 或 @虾局长 才触发
- 适合真实平台映射
- 适合解释“真实接入约束”

### Preset B：平衡模式（默认）
- 虾局长可以基于聊天生成“建议气泡”
- 真正发卡或发倡议前，仍由一个用户确认
- 最适合比赛 demo

### Preset C：主动主持模式
- 当置信度足够高时，虾局长可以直接发“建议卡”或“倡议卡”
- 适合展示更强的 AI 官方 Agent 形态
- 但需要在卡片说明里体现“基于最近聊天生成”

### Preset D：冲突安全模式
- 冲突检测可以自动发生
- 但公开桥接前优先给一个“是否由我介入”的确认步骤
- 避免显得越界

## 3. 功能级别触发矩阵

| 功能 | 默认触发方式 | 是否允许自动建议 | 是否允许无需用户操作直接发出 | 是否显示触发人 |
|---|---|---:|---:|---|
| 收口组局 | 建议 chip / @虾局长 | 是 | 在主动主持模式下可直接发建议卡 | 普通模式显示；匿名模式可隐藏 |
| 匿名倡议 | 用户点击“匿名发起” | 否，先建议后确认 | 由用户确认后可直接群发 | 不显示真实发起人 |
| 时间/地点投票 | 成局卡按钮 | 否 | 成局已建立后可自动更新进展 | 不强调触发人 |
| 提醒未回复 | 投票进展卡按钮 | 是 | 成局后在录屏模式可自动触发 | 不强调触发人 |
| 确认成局 | 票数充分 / 管理员确认 / 自动阈值 | 是 | 是 | 不强调触发人 |
| 冲突桥梁 | 建议 chip / @虾局长 / 自动识别 | 是 | 仅在安全模式确认后自动公开桥接 | 默认不显示是谁求助；studio 可查看 |
| 活动回忆卡 | 活动结束后自动生成 | 否 | 是 | 无需显示 |
| 游戏组队建议 | 建议 chip / 自动识别聊天中的开黑意图 | 是 | 在主动主持模式下可直接发建议卡 | 可隐藏具体触发人 |
| 游戏回忆卡 | 对局结束或战绩导入后 | 否 | 是 | 无需显示 |

## 4. 触发人展示规则

### 普通触发
展示方式：
- 用户消息出现在群里：
  - `@虾局长 帮我收口这局`
- 这类场景让评委理解：
  - 用户主动召唤 Agent
  - 更贴近 QQ 平台现实

### 匿名触发
展示方式：
- 群里不出现真实用户消息
- 直接出现虾局长发出的倡议卡 / 倡议消息
- 文字示例：
  - `虾局长建议：有人想在周五一起吃烤肉，要不要我来帮大家收口？`

### 自动建议
展示方式：
- 虾局长先发一条轻量提示
- 或者在输入区上方出现建议 chips：
  - `帮我收口这局`
  - `匿名代发`
  - `发起时间投票`

### 自动后续动作
典型包括：
- 投票结果更新
- 未回复提醒
- 确认成局
- 活动回忆卡
- 游戏回顾

这些动作不需要再强调“是谁点的”，因为重点已经从“谁触发”转到“虾局长正在承担组织责任”。

## 5. 功能细节定义

## 5.1 智能收口组局

### 问题
群里已经有兴趣，但没有人愿意做组织者。

### demo 中的理想链路
1. 群里自然聊天铺垫
2. 虾局长出现建议 chip 或直接建议泡泡
3. 用户选择：
   - `@虾局长 帮我收口这局`
4. 虾局长生成成局卡
5. 时间 / 地点 / 参加意向推进
6. 更新投票进展
7. 提醒未回复
8. 确认成局
9. 生成回忆卡

### LLM 使用点
- 从最近消息中提取：
  - 活动类型
  - 时间候选
  - 地点候选
  - 参与意愿
  - 未确认信息
- 生成 bot 建议文案
- 生成卡片摘要

### UI 表现
- 成局卡
- 投票进展卡
- 确认成局卡
- 回忆卡

## 5.2 匿名倡议

### 问题
有些人想提议，但不想暴露自己来承担责任。

### demo 中的理想链路
1. 群里有模糊想法
2. 评委点击：
   - `匿名发起一个倡议`
3. 虾局长展示一个极简确认浮层：
   - 你想让大家做什么？
   - 要不要顺便发起投票？
4. 虾局长以官方口吻代发倡议
5. 群内开始响应
6. 可进一步升级成投票或成局

### LLM 使用点
- 把用户私下表达转换成群里更自然、安全的倡议文案
- 决定措辞语气
- 生成更不容易尴尬的投票选项

### 注意
匿名场景里，群成员不能看到是谁触发的。
但 Studio Mode 允许看到内部 trace，方便讲解。

## 5.3 冲突桥梁

### 问题
群里吵起来时，传统群管理手段只有禁言、踢人、放任不管，太粗。

### demo 中的理想链路
1. 群里出现来回争执
2. 虾局长识别到冲突升级
3. 先给一个“是否需要我帮忙转述”的建议入口
4. 用户选择：
   - `让虾局长来转述`
5. 虾局长发布桥接卡：
   - 双方真正关心的点
   - 共识 / 分歧
   - 可选动作：
     - 冷静 10 分钟
     - 继续私聊
     - 由虾局长转述
     - 先回到要解决的问题
6. 输出一条中性、温和的桥接消息

### LLM 使用点
- 提取双方诉求
- 去攻击性改写
- 生成中性转述
- 生成降温建议

### 注意
不要承诺“AI 一定解决冲突”。
产品定位是：**比什么都不做更好**，给群管理新增一套温和工具。

## 5.4 腾讯游戏局

### 问题
群里经常想开黑、组队、打活动，但仍有相同的组织问题：
- 谁先提
- 谁能来
- 打什么模式
- 活动结束后没人整理高光

### demo 中的理想链路（王者荣耀）
1. 群里有人说今晚五排吗
2. 虾局长给出建议：
   - `要不要我来帮你们收口开黑局`
3. 生成游戏组队卡：
   - 时间候选
   - 差几个位置
   - 是否接受补位
4. 开打后或假定结束后
5. 生成战绩 / 高光回忆卡：
   - MVP
   - 高光时刻
   - 下次再约

### LLM 使用点
- 识别开黑/组队意图
- 生成组队说明与补位文案
- 生成游戏风格回顾文案

## 6. 最终产品的主线与支线怎么组织

### 不删功能，但分层展示

#### 主线（必须录进视频）
- 周五烤肉局

#### 支线 1（现场 20~30 秒）
- 匿名倡议

#### 支线 2（现场 20~30 秒）
- 冲突桥梁

#### 支线 3（现场 20~30 秒）
- 王者荣耀五排组队与回忆

这样做的目的不是弱化功能，而是：
- 让评委先建立主心智
- 再快速看到能力外延
- 形成“这个产品不是单点工具，而是一个官方社交 Agent”的认知
