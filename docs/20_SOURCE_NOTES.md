# 20 数据与来源笔记

以下资料用于支持产品文档中的问题定义、用户洞察背景与平台可行性说明。

## 1. 社交与即时通信高频
建议使用：
- CNNIC《第55次中国互联网络发展状况统计报告》
  - 即时通信用户规模：10.81 亿
  - 社交网络用户规模：11.01 亿
  - 统计时点：2024 年 12 月

用途：
- 证明群聊是高频基础设施
- 证明“群内活动组织”不是边缘场景

## 2. 校园群体规模
建议使用：
- 《2024年全国教育事业发展统计公报》
  - 全国各种形式高等教育在学总规模：4846.00 万人

用途：
- 证明高校学生不是小众场景
- 证明校园社交 / 群聊 / 组局具备规模基础

## 3. QQ 仍是大关系链容器
建议使用：
- 腾讯 2025 年全年业绩披露
  - QQ 移动终端月活跃账户数：5.08 亿

用途：
- 证明 QQ 仍然值得作为产品实战场
- 说明题目 3 的平台基础仍然足够大

## 4. QQ 机器人真实平台能力
建议使用：
- QQ 机器人官方文档
  - 机器人可在群聊 / 频道互动
  - 用户可与机器人单独对话
  - 使用 AppID / AppSecret 与 Access Token
  - 支持事件订阅与消息发送
  - 主动推送能力已发生策略变化，真实产品不应强依赖主动 push

用途：
- 说明 demo 虽是 QQ-like，但真实落地有路径
- 解释为什么要区分自动建议与显式 @ 触发

## 5. 腾讯生态与游戏扩展
建议使用：
- QQ 应用商店描述中强调群消息、社交、游戏中心等能力
- 结合腾讯游戏生态，说明王者荣耀、洛克王国等游戏局扩展并非无关幻想，而是 QQ 语境内可理解的社交延展

## 6. 如何写进 PDF

### 推荐写法
不要把数据堆成报告。
建议这样写：

> 虾局长瞄准的是一个“高频 × 高痛”的群聊组织场景。即时通信与社交网络本身就是超高频基础设施；高校学生又是群聊组织密度极高的人群；而 QQ 作为仍拥有大规模移动端活跃账户的平台，天然适合承接“从讨论到成局、从活动到回忆”的官方 Agent 形态。

## 7. 一级研究补充建议
如果时间允许，建议补充：
- 5~8 位学生访谈
- 20~50 份问卷
- 2~3 个真实聊天样本复盘

一级研究问题建议：
- 你最近一次在群里发起活动是什么？
- 你会不会不想当组织者？
- 你为什么不想第一个说？
- 你觉得群里最烦的组织问题是什么？
- 你能接受官方 Agent 匿名替你发起倡议吗？
- 你觉得吵架时群主除了禁言还有更好的办法吗？
- 你会不会希望活动结束后自动有个回忆卡？

## 8. 使用方式
本文件是“写文档时的证据池”，不是最终 PDF 文本。
写 PDF 时请：

- 数据要少而准
