# 21 最终验收清单

## A. 产品定义

- [ ] 项目仍明确是题目 3：QQ 养虾官方 Agent / 社交新玩法
- [ ] 产品不是通用聊天机器人
- [ ] 四条能力线都已进入正式产品范围：
  - [ ] 收口组局
  - [ ] 匿名倡议
  - [ ] 冲突桥梁
  - [ ] 腾讯游戏局 / 回忆

## B. 评委体验

- [ ] 无 LLM 评审模式默认体验清晰
- [ ] 首屏 10 秒内能看懂项目
- [ ] 主线 3 分钟内可以讲完
- [ ] 支线可在 20~30 秒内单独展示
- [ ] 评委不需要自由输入也能体验核心价值
- [ ] 没有开发控制台污染评委界面

## C. 讲解体验

- [ ] `/judge` 中有清晰返回主界面和进入真实 LLM 工作台的入口
- [ ] 可一键逐条演示
- [ ] 每一幕都有痛点、设计意图和期望效果说明
- [ ] 可一键 replay
- [ ] 讲解时 UI 干净

## D. 功能完整性

### 收口组局
- [ ] 能从群聊意图进入成局建议
- [ ] 有时间 / 地点 / 参加意向推进
- [ ] 有未回复提醒
- [ ] 有确认成局
- [ ] 有局后回忆卡

### 匿名倡议
- [ ] 有明确入口
- [ ] 不暴露真实发起人
- [ ] 能被虾局长代发到群里
- [ ] 能进一步推进成投票 / 成局

### 冲突桥梁
- [ ] 有冲突样例
- [ ] 能总结双方诉求
- [ ] 能给出中性转述
- [ ] 有后续选项卡

### 腾讯游戏局
- [ ] 至少一个王者荣耀场景
- [ ] 能体现组队推进
- [ ] 能体现局后回顾 / 下次再约

## E. LLM 真实性

- [ ] 存在真实 LLM 接入
- [ ] 前端不直接持有密钥
- [ ] 使用 schema / Zod 校验
- [ ] 支持 mock / live / snapshot
- [ ] live 失败可回退
- [ ] 真实 LLM 工作台可保存 snapshot
- [ ] 至少演示以下 live 任务中的 2 个：
  - [ ] 意图识别
  - [ ] 匿名倡议改写
  - [ ] 冲突桥梁
  - [ ] 回忆卡生成
  - [ ] 游戏回顾生成

## F. 技术结构

- [ ] 场景引擎已从粗 step 升级为 beat/timeline
- [ ] 卡片按钮有显式 actionId
- [ ] autoplay 不依赖硬编码 actionId 列表
- [ ] 无 LLM 评审模式与真实 LLM 工作台分离
- [ ] fixtures 与组件分层清楚

## G. 部署

- [ ] 有外网可访问链接
- [ ] judge route 稳定
- [ ] studio route 有访问控制
- [ ] 环境变量已配置
- [ ] 有 fallback 方案

- [ ] 它是一个完整的比赛作品，而不是半成品前端 demo
