# 22 录屏模式与演示模式说明

## 1. 为什么必须单独做 Recording Mode

普通 autoplay 只适合自己看。
比赛视频录制需要的是：
- 节奏稳定
- 讲解点明确
- 可暂停
- 可重复
- 不被临时点击打乱

所以 recording mode 不是“快一点的 autoplay”，而是 **内置导演节拍器**。

## 2. 模式区分

### Judge Mode
- 给评委亲手点
- 控制少
- 引导强

### Recording Mode
- 给你录视频
- 自动跑主线
- 在关键点暂停
- 不暴露 studio UI

### Studio Mode
- 给你调试
- 可插入 live LLM
- 可切支线
- 可存快照

## 3. Recording Mode 的主线结构

建议固定为 6 幕：

### 幕 1：群聊自然起势
目的：
- 先让观众看见真实问题，而不是先看功能说明

### 幕 2：虾局长介入
目的：
- 告诉观众：这不是投票插件，而是会收口的 Agent

### 幕 3：投票推进
目的：
- 告诉观众：虾局长在处理“我都可以”和未回复问题

### 幕 4：确认成局
目的：
- 告诉观众：虾局长承担组织动作

### 幕 5：局后回忆
目的：
- 告诉观众：它会把一次活动沉淀成可复用资产

### 幕 6：支线闪示
目的：
- 告诉观众：它不只有吃饭局，还能匿名倡议、桥接冲突、服务游戏局

## 4. 建议的自动暂停点

### Pause 1
成局卡出现后暂停 2~4 秒  
你可以讲：
- 不想当组织者
- 虾局长替人承担收口责任

### Pause 2
投票进展卡出现后暂停 2~4 秒  
你可以讲：
- “我都可以”如何被结构化处理
- 未回复提醒为什么重要

### Pause 3
确认成局卡出现后暂停 2~4 秒  
你可以讲：
- 它已经不只是投票，而是在推进活动落地

### Pause 4
回忆卡出现后暂停 2~4 秒  
你可以讲：
- 关系资产沉淀
- 下次再约更容易发生

### Pause 5
匿名倡议 / 冲突桥梁 / 游戏局闪示时各暂停 1.5~2 秒  
你可以快速讲：
- 不想背组织责任
- 禁言之外的管理新手段
- 腾讯生态里的游戏社交延展

## 5. Recording Mode 需要的控制

对录屏者可见的小浮层即可：

- 开始
- 暂停
- 下一幕
- Replay
- 显示/隐藏字幕提示（可选）

不要显示：
- 场景树
- debug 数据
- mode pill
- 原始 JSON

## 6. 最佳录屏时长分配

### 0:00 - 0:20
问题导入 + 群聊起势

### 0:20 - 1:20
收口组局主线推进

### 1:20 - 2:00
投票 → 确认成局

### 2:00 - 2:25
回忆卡

### 2:25 - 2:55
匿名倡议 / 冲突桥梁 / 游戏局闪示

### 2:55 - 3:00
结尾一句总结

## 7. 录屏模式对工程的具体要求

- timeline 支持 pause point
- 每个 beat 可配置最短停留时长
- 可自动继续，也可手动继续
- 可记录“当前录到哪一幕”
- 支持 replay 到指定幕

## 8. 切记

你不是在录一个“可以随便点的网页”，而是在录一个 **产品故事**。
Recording Mode 的目标不是自由，而是 **稳定、好讲、好看、好过评审**。
