# 23 比赛对齐说明（题目 3）

## 1. 当前已确认的官方赛题包结构
根据官方赛题包可见的 tab，比赛信息至少包含以下板块：
- 目录&导航
- 初赛提交指南
- 赛题-命题赛道
- 赛题-开放赛道
- 赛程&奖项说明
- 评分细则
- FAQ 常见问题

这意味着仓库内容不应该只覆盖“代码怎么跑”，还要覆盖：
- 提交前要交什么
- 赛题到底是什么
- 评审关注什么
- 现场 Demo 怎么帮助评委理解

## 2. 本项目只关注命题赛道题目 3
题目 3 的核心方向是：
- QQ
- 养虾
- 社交新玩法
- AI 产品能力，而不是单点模型炫技

所以本仓库所有内容都应该围绕：
- 群聊
- 社交推进
- 组织责任转移
- 匿名表达
- 冲突桥梁
- 回忆沉淀
来展开。

## 3. 仓库如何对齐比赛结构

### 对齐“初赛提交指南”
由 `submission/` 目录承接：
- Judge 说明页
- PDF 大纲
- 视频脚本
- 截图位点
- 材料清单

### 对齐“赛题-命题赛道”
由 `docs/14`、`docs/15` 承接：
- 产品定义
- 能力矩阵
- 触发规则
- 场景设计

### 对齐“评分细则”
由 `docs/21` 承接：
- 赛道适配
- 完整性
- 创新性
- 用户洞察
- AI 原生性
- 落地与商业扩展

### 对齐“FAQ”
由 `无 LLM 评审模式 / 真实 LLM 工作台` 承接：
- 让评委不迷路
- 让现场讲解不翻车
- 让产品逻辑可被快速理解

## 4. 最终目标
不是“做一个网页”，而是把仓库本身变成：
- 一个可运行 demo
- 一个可录视频的舞台
- 一个可快速抽取比赛材料的素材库
- 一个可向评委解释完整产品逻辑的证据仓
